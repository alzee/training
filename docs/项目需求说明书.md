# 民兵军事训练信息化管理系统 项目需求说明书

版本：v1.0（草案）  
日期：2025-11-08  
密级：内部/受控  
编制单位：项目组

---

## 1. 背景与建设动因

### 1.1 背景
民兵力量作为国防后备的重要组成部分，承担战备支援、应急救援、灾害处置、社会稳定等多维任务。现有训练管理方式在计划编制、编组调度、课程资源、考核评估、数据记录与复盘分析等环节仍存在分散、低效、缺乏量化、难以追溯的问题，难以满足新形势下“精准训练、实时掌控、智能评估、快速动员”的要求。为推进标准化、数字化、智能化建设，提高战备生成与应急响应质量与速度，拟建设“民兵军事训练信息化管理系统”。

### 1.2 建设动因
- 训练计划与执行缺乏全周期数据支撑，资源排程存在冲突与闲置。
- 个人与编组能力画像缺乏动态更新，难以形成差异化训练策略。
- 考核评估多为线下手工记录，数据碎片化、滞后与不可复用。
- 装备与场地使用缺乏统筹，维护计划与训练节奏脱节。
- 缺少对训效趋势、风险点（安全/伤害/装备故障）的预测预警机制。
- 训练成果与战备动员关联度弱，难以快速形成任务匹配与临战调度。

### 1.3 建设目标
- 打造覆盖“计划制定→编组排程→资源保障→课程实施→成绩采集→评估分析→复盘提升→战备应用”全生命周期的统一平台。
- 建立个人与编组多维能力画像，实现个性化与针对性训练推荐。
- 引入智能化与数据分析，实现训练资源优化调度、训练风险预警与模型化评估。
- 形成标准化数据模型与主数据体系，支撑跨区域、跨单位数据互认与汇总分析。
- 提供可追溯、可度量的训练成效指标体系，为战备动员与应急支援提供决策依据。
- 满足分级保密与审计要求，确保数据安全、访问合规与操作留痕。

### 1.4 成功标准（示例）
- 训练计划冲突率显著下降（≤ 设定阈值）。
- 课程资源利用率提升（≥ 目标百分比）。
- 训练成绩采集准确率与及时率达到目标。
- 个性化推荐采纳率提升，人员能力提升曲线可量化。
- 关键风险预警准确率达到模型阈值，误报/漏报率可控。
- 数据审计与合规检查通过设定安全评估等级。

## 2. 系统定位与范围

### 2.1 定位
面向多层级民兵组织的“训练计划管理 + 编组编排 + 课程资源库 + 训练执行与数据采集 + 评估分析 + 智能推荐 + 战备支撑”综合管理平台，作为指挥调度与训练建设的统一数字底座。支持平时训练与战备转化的无缝衔接，强化数据闭环与持续改进。

### 2.2 范围（业务域）
- 训练计划与年度/季度/专项方案管理
- 编组与人员管理（编成结构、资质、出勤、轮换）
- 课程与科目库（标准化大纲、教学资源、适配条件）
- 训练排程与场地装备调度
- 训练执行记录与实时数据采集（成绩、行为、装备状态、考勤）
- 考核与评估（个人/编组/单位多维指标）
- 训练效果分析与复盘（趋势、对标、差异化短板）
- 智能推荐与能力画像（个性化训练路径、重难点强化）
- 风险预警与安全管理（伤害、装备故障、执行偏差）
- 战备支撑（任务匹配、快速动员、能力适配）
- 报表与档案归档（训练档案、评估报告、对比分析）
- 权限、审计、安全与合规

### 2.3 不在本期范围（示例）
- 外部专有视频侦察系统深度融合（保留标准接口接入能力）。
- 高仿真大型模拟训练硬件集成（仅预留模拟接口规范）。

## 3. 用户角色与典型场景

| 角色 | 主要诉求 | 示例场景 |
|------|-----------|---------|
| 指挥与主管 | 训练态势、能力评估、资源统筹、战备转换 | 查看单位训练完成率与差异短板，审批专项训练计划 |
| 训练管理员 | 计划编制、排程冲突处理、课程配置 | 年度计划拆分到季度、生成周排程并解决场地冲突 |
| 教官/教练员 | 课程实施、成绩记录、问题反馈 | 推送课程大纲、采集学员动作评分与出勤异常 |
| 民兵人员 | 个人训练任务、课程资源、自主学习 | 接收个性化强化训练与短板提示 |
| 评估与考核员 | 成绩审核、评估报告、对标分析 | 审核考核成绩，生成单位能力提升趋势报告 |
| 装备与场地管理员 | 使用调度、维护计划、故障预警 | 场地占用视图、装备保养到期提示 |
| 安全与医务保障 | 风险监测、伤害记录、应急预案 | 高强度训练风险提示与伤害统计溯源 |
| 运维与安全审计 | 权限配置、日志审计、安全基线 | 检索操作日志与访问行为分析 |

典型场景：
1. 年度训练计划拆解为季度与专项任务 → 自动生成排程草案 → 冲突检测与资源调优。
2. 教官发起课程实施 → 人员签到 → 实时成绩/传感数据采集 → 自动能力画像更新。
3. 系统预警某编组体能训练偏差趋势 → 推荐强化科目与补强周期。
4. 装备故障高频 → 系统生成维护优先级与替代资源方案。
5. 临战动员查询：按任务需求快速匹配最适配编组（技能/训练完成度/可用状态）。

## 4. 功能需求分解

### 4.1 训练计划管理
- 多层级计划：年度→季度→月/周→专项；可视化层级结构。
- 模板与版本：计划模板库、版本对比与回滚。
- 冲突检测：时间、场地、教官、装备冲突自动标识与替代建议。
- 依赖与约束：先决科目、体能等级门槛、资质限制策略化配置。

### 4.2 编组与人员管理
- 编组结构：层级编成、临时任务编组、历史演变链路。
- 人员档案：基础信息、资质证书、训练履历、健康与伤害记录。
- 出勤与状态：签到、异常（缺勤/早退/伤情）、出勤统计与趋势分析。
- 人员轮换与补员：规则化策略（疲劳度、训练负荷、任务适配）。

### 4.3 课程与科目资源库
- 课程标准：科目目标、时间、器材、评分标准、健康风险点。
- 适配条件：人员等级、前置科目、场地/装备需求。
- 教学资源：文档、图示、视频、动作分解、模拟场景脚本。
- 版本迭代：修订记录、适用时间窗、评估审批流程。

### 4.4 排程与资源调度
- 排程引擎：基于约束（场地容量、教官负荷、装备可用性）生成候选方案。
- 优化策略：最小冲突、最大利用率、平滑训练负荷。
- 手工调整：交互式拖拽与冲突实时提示。
- 场地与装备视图：日/周/月占用热力图与可用窗口。

### 4.5 训练执行与采集
- 签到与身份确认（多方式）。
- 成绩录入：手工/批量/设备自动采集（传感器、计时、动作识别）。
- 异常事件记录：伤害、装备故障、违规操作、超负荷提醒。
- 行为数据：动作质量、完成度、心率/体能（可选传感源）。

### 4.6 考核与评估
- 考核计划：集中考核/日常考核/补考机制。
- 评分模型：标准分、难度修正、权重与综合能力指数。
- 审核流程：初评→复核→归档，留痕与差异分析。
- 对标分析：同岗位/同编组/跨区域能力对比与分位数统计。

### 4.7 能力画像与智能推荐
- 画像维度：体能、技能、战术、协同、心理、稳定性（出勤与波动）。
- 短板识别：偏差检测、训练缺口与趋势下滑预警。
- 推荐引擎：差异化课程、补强计划、负荷调节与恢复建议。
- 成效跟踪：推荐采纳率与效果提升曲线。

### 4.8 风险与安全管理
- 风险库：常见伤害、设备故障模式、超负荷警戒阈值。
- 预测模型：基于历史负荷、当前排程、个体状态生成风险评分。
- 预警分级：提示/预警/警报；通知路径与处置记录。
- 安全事件复盘：原因分类、链路分析与整改闭环。

### 4.9 装备与场地保障
- 台账与状态：装备寿命、保养、故障登记、场地特性与容量。
- 使用调度：预约、冲突检测、维护窗口避让。
- 维护与保养计划：周期自动生成与提醒，优先级依据故障趋势。
- 备份与替代资源：故障与短缺时的自动替代方案建议。

### 4.10 战备支撑与动员匹配
- 任务需求建模：技能/人数/等级/训练完成度/健康状态等参数。
- 匹配引擎：推荐最适编组或人员清单，包含可用时间窗与风险提示。
- 模拟演练：动员时间估算、能力缺口分析、补强建议。
- 任务准备度指数：综合训练完成度、近期表现、风险指标计算。

### 4.11 报表与档案
- 训练完成率、课程利用、装备使用、伤害统计、能力提升趋势。
- 自定义报表与模板；计划与执行偏差分析。
- 档案归档：版本化封存、检索与审计留痕。
- 智能摘要：关键变化、异常点、短板趋势自动生成。

### 4.12 权限与审计
- 分级访问：角色/任务/数据域三维控制。
- 最小权限与临时授权；高敏操作二次确认。
- 审计日志：计划变更、成绩修改、推荐采纳、预警处理全量留痕。
- 异常访问行为检测与告警。

## 5. AI 与智能化模块

### 5.1 模块清单
- 训练排程优化器：多约束（教官、场地、装备、负荷）生成最优方案。
- 能力画像引擎：融合多源数据构建动态多维能力向量。
- 个性化推荐模型：课程、补强、恢复、节奏调节建议。
- 风险预测模型：伤害/故障/过载趋势与风险分值。
- 任务匹配模型：战备需求与人员/编组能力快速匹配。
- 短板发现与聚类分析：群体共性问题与结构性缺口识别。
- 训练效果归因分析：提升或下滑的关键要因挖掘。
- 语义问答与指令解析：自然语言查询训练态势与生成报告。
- 数字孪生与模拟（可选）：模拟负荷变化对能力与风险的影响。

### 5.2 治理与质量
- 模型指标：准确率、召回率、F1、误报/漏报、稳定性（漂移检测）。
- 版本与回滚：上线审批、试运行、异常自动降级。
- 可解释性：关键推荐/预警附特征贡献与可复现实验记录。
- 人工复核：高风险输出强制人工确认；人工反馈回流再训练。

### 5.3 数据要求
- 数据标准：时间戳、来源、单位、编码一致性。
- 标注规范：成绩类型、风险事件分类、伤害类型、装备故障编码。
- 清洗与补全：缺失值、异常值、重复记录自动检测与处置。
- 隐私与分级：敏感个人信息脱敏策略、访问分级与审计。

### 5.4 运行要求
- 近实时：风险预警与排程冲突检测具备可配置低延迟目标。
- 伸缩性：模型推理服务可按负载水平扩展。
- 异常防护：输入校验、对抗样本初步侦测（动作/传感数据异常）。

## 6. 安全与保密要求
- 信息分级：个人敏感、训练成绩、战备评估等分类与访问域隔离。
- 身份与认证：多因素、会话有效期控制、异常行为锁定。
- 授权策略：角色+属性（编组/任务期）+风险级别的多维访问控制。
- 审计与合规：操作、数据访问、模型推理与推荐采纳均留痕；完整性校验。
- 数据加密与脱敏：传输与静态加密；训练分析使用脱敏字段；战备映射隔离层。
- 供应链与引入组件审查：第三方数据/模型/设备接入评估与白名单。
- 安全预警：权限滥用、数据批量导出、异常频次访问检测。

---

## 7. 数据治理与主数据体系
- 主数据域：人员、编组、课程科目、装备、场地、考核项目、风险事件类型。
- 数据字典：字段定义、取值范围、计量单位、更新频率与责任人。
- 质量指标：完备率、及时率、准确率、重复率、漂移率（模型训练数据）。
- 数据血缘：从采集（传感/录入）到分析/推荐的全链路追踪。
- 生命周期：采集→加工→使用→归档→销毁；到期自动提醒。
- 交换与接口：标准结构、版本、错误码、流控策略与审计。

---

## 8. 性能、可用性与运维
- 可用性目标：平台总体≥既定阈值；关键排程与预警服务更高SLO。
- 响应性能：常规查询与录入界面流畅；冲突检测与推荐具备限定最大延迟。
- 容量规划：按人员规模、课程数、并发场次、数据采集频次进行基线测算。
- 异常与降级：模型不可用时回退规则算法；推理超时采用缓存或兜底策略。
- 监控与可观测：指标（吞吐/延迟/错误率）、日志、分布式追踪、事件告警。
- 灾备与演练：定期演练排程恢复、数据回滚、权限隔离故障注入测试。

---

## 9. 部署与环境管理
- 环境分层：规划开发、联调、测试、预生产、生产与灾备。
- 分域部署：核心战备评估与训练日常分析域隔离；必要数据桥接审计。
- 弱网与离线：现场训练弱网缓存与回传机制；断连重试与数据一致性校验。
- 变更管理：版本审批、灰度验证、紧急回滚与稽核报告。

---

## 10. 测试与验收
- 覆盖层次：单元、集成、场景模拟（训练日程/高并发签到）、性能、安全、数据质量、AI 专项、UAT。
- AI 专项测试：训练推荐有效性（提升率）、预警准确性（ROC/AUC）、漂移监控（特征分布变化）。
- 验收指标：功能通过率、性能峰值指标、数据完整性、审计一致性、AI 模型基准阈值、风险预警误报率。
- 交付物：测试报告、指标对照表、缺陷清单与修复确认、模型评估档案。

## 11. 里程碑规划（示例）
- M0 立项与范围确认
- M1 需求冻结与原型评审
- M2 数据模型与主数据建模、计划/编组/课程基础能力实现
- M3 排程优化与成绩采集、评估初版上线
- M4 AI 推荐与风险预警试运行、画像引擎落地
- M5 战备匹配与复盘分析、报表与档案模块完善
- M6 安全评估与性能压力测试、演练与灾备验证
- M7 试运行与优化迭代、验收与移交

## 12. 风险识别与缓解
- 历史数据缺失：启动前置数据补录与最小可用数据集策略。
- 推荐/预警效果不佳：人机协同标注、持续再训练与A/B评估。
- 训练现场弱网：离线缓存与延迟同步；关键数据优先上传队列。
- 隐私与敏感信息泄露：分级脱敏、访问审计与异常行为告警。
- 资源排程复杂度高：迭代排程优化器与人工调节辅助界面。

---

## 13. KPI 指标（示例）
- 训练计划冲突解决率 ≥ 既定百分比
- 课程资源使用率 ≥ 既定阈值
- 成绩采集及时率 ≥ 既定阈值
- 个性化推荐采纳率 ≥ 既定阈值，提升幅度可量化
- 关键风险预警准确率 ≥ 模型基线；误报/漏报 ≤ 阈值
- 战备任务匹配响应时间 ≤ 设定秒数
- 数据质量指标（完备/准确/重复）全部达到基线
- 审计日志完整性校验通过率 100%

## 15. 术语与定义（节选）
- 编组：任务或训练单位的组织结构单元。
- 能力画像：基于多维训练与行为数据构建的人员或编组能力特征集合。
- 补强计划：针对能力短板与风险提示生成的训练强化方案。
- 风险预警：预测可能出现的伤害、故障或训练偏差并分级提示的机制。
- 战备匹配：将任务需求与人员/编组能力快速匹配的过程。

## 16. 变更与配置管理要点
- 需求变更流程：提出→评估影响→优先级调整→审批→实施→回顾。
- 配置项：评分权重、风险阈值、推荐算法策略、排程约束参数版本化。
- 审计链：变更记录与执行结果留痕，可回溯验证。
